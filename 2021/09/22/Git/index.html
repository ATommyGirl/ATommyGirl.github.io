Git 的各种骚操作

YYLittleCat
2021-09-22
7788

git 是一个分散式版本控制软件。最初目的是为更好地管理 Linux 内核开发而设计。

什么是版本控制？

什么是“版本控制”，你为什么要关心？版本控制是一种系统，它记录一段时间内对一个文件或一组文件的更改，以便您以后可以调用特定版本。对于开发来讲，多数以软件源代码作为受版本控制的文件，但实际上几乎可以对计算机上的任何类型的文件执行此操作。

常见的版本控制系统：

  • 本地版本控制系统 - VCS，例如 RCS
  • 集中版本控制系统 - CVCS，例如 CVS、Subversion、Perforce
  • 分布式版本控制系统 - DVCS，例如 Git、Mercurial、Bazaar 或 Darcs

今天的主题 Git 就属于 DVCS，在 DVCS 中，客户端不仅仅查看文件的最新快照，相反，它们完全镜像存储库，包括其完整历史记录。因此，如果任何服务器死机，这些系统通过该服务器进行协作，则可以将任何客户端存储库复制回服务器以恢复它。每个克隆实际上都是所有数据的完整备份。

Git 简史

与生活中的许多伟大事物一样，Git 始于一些创造性的破坏和激烈的争议。

Linux 内核是一个规模相当大的开源软件项目。在 Linux 内核维护的早期（1991-2002 年），对软件的更改以补丁和存档文件的形式传递。2002 年，Linux 内核项目开始使用名为 BitKeeper 的专有 DVCS。

2005 年，开发 Linux 内核的社区与开发 BitKeeper 的商业公司之间的关系破裂，该工具的免费状态被取消。这促使 Linux 开发社区（尤其是 Linux 的创建者 Linus Torvalds）根据他们在使用 BitKeeper 时学到的一些经验教训开发自己的工具。新系统的一些目标如下：

  • 速度
  • 简单的设计
  • 对非线性开发的强力支持（数千个并行分支）
  • 完全分布式
  • 能够高效地处理像 Linux 内核这样的大型项目（速度和数据大小）

自 2005 年诞生以来，Git 已经发展并成熟为易于使用并保留了这些最初的品质。它非常快，对于大型项目非常有效，并且它具有用于非线性开发的令人难以置信的分支系统。
